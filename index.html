DB commands

show dbs -->list the database present in connected cluster

use db_name (eg: use sample_airbnb) --> switch to particular db

db (or) db.getName() -->get the db_name of connected db

db.createCollection("collection_name") -->will create a collection in connected db with given name

db.dropDatabase() -->will delete the connected db

collection commands

show collections -->will return all collections present in connected db

db.collection_name.drop() --> will delete the collection

db.collection_name.findOne() --> will return the first document present in that specified collection

db.collection_name.find() -->will return first 20 documents and with it command see next 20 documents and so on

db.collection_name.find().limit(numeric_value)-->return only numeric_value of records.

db.collection_name.find().skip(numeric_value)-->skip the first numeric_value records and return remaining records

db.collection_name.find().limit(numeric_value).skip(numeric_value)-->skip the first numeric_value records and return numeric_value records

CRUD operations

Note: condition must be single object with field-value pair

db.collection_name.insertOne({object}) -->will insert one document to the collection

db.collection_name.insertMany([{object1},{object2}]) -->will insert multiple documents to the collection. Here [] it denotes array. insertMany function accepts array of objects

db.collection_name.deleteOne({condition})-->will delete one document whenever the first matches is true.

db.collection_name.deleteMany({condition})-->will delete multiple documents which matches the given condition

db.collection_name.updateOne({condition},{$set:{field:value}}) --> will update one record which matches the given condition, if the field is present already it will update its corresponding value else it will add new field-value pair in that record

db.collection_name.updateMany({condition},{$set:{field:value}}) --> will update all record which matches the given condition, if the field is present already it will update its corresponding value else it will add new field-value pair in the records

db.collection_name.updateOne({condition},{$inc:{field:value}})-->will increase the field value by defined value for first record which matches the condition.

db.collection_name.updateMany({condition},{$inc:{field:value}})-->will increase the field value by defined value for all records which matches the condition.

db.collection_name.updateOne({condition},{$push:{field:value}})-->will push the given value to array field, here the field must be already present and it should an array type

operators

comparison operators

eq -->equal operator usage: {field:{$eq:"value"}}

ne -->not equal usage: {field:{$ne:"value"}}

gt-->greater than usage: {field:{$gt:value}}

gte--> greater than or equal to usage: {field:{$gte:value}}

lt -->less than

lte -->less than or equal to

in--> used to match any value in the given array usage: {field:{$in:["value1","value2","value3",......, so on]}}

nin --> used to match other value in the given array usage: {field:{$nin:["value1","value2",....,so on]}}

logical operators

and --> returns records when all given condition is true usage: {$and:[{condition1},{condition2},...,so on]}

or -->returns records when any one condition is true usage: {$or:[{condition1},{condition2},...,so on]}

nor --> returns records when all condition is false usage: {$nor:[{condition1},{condition2},...,so on]}

not --> inverse the conditions specified usage: {$not:{condition}}

array query operators

all --> returns records when given array matches without checking orders usage: {field:{$all:["value1","value2",...,so on]}}

elemMatch--> returns records with conditions in the array usage: {field:{$elemMatch:{condition1,condition2}}}

size --> returns array which having the given size(length) usage: {field:{$size:value}}
